# Слайд 3: Счётчик очков (TextSprite)
В `__init__` создаём текстовый спрайт для отображения счёта:

```python
# 4. Счёт
self.score = 0
self.score_text = s.TextSprite(
    f"Счёт: {self.score}", pos=(s.WH_C[0], 50),
    font_size=40, color=(0, 0, 0), scene=self, sorting_order=20
)
```

`TextSprite` — это спрайт-текст. Он рисуется прямо на экране, как HUD в настоящих играх.

# Слайд 4: Логика подсчёта
Как понять, что птичка пролетела мимо трубы? Если координата X трубы стала **меньше** координаты X птички — труба позади!

При создании нижней трубы добавляем флаг:
```python
bottom.passed = False  # Флаг для подсчёта очков
```

В `update` проверяем:
```python
# --- ПОДСЧЁТ ОЧКОВ ---
for pipe in self.pipes:
    if hasattr(pipe, 'passed') and not pipe.passed and pipe.x < self.player.x:
        pipe.passed = True
        self.score += 1
        self.score_text.text = f"Счёт: {self.score}"
```

**Почему только нижняя труба?** Мы создаём пару — верхнюю и нижнюю. Если считать обе, за одно препятствие получим 2 очка. А нам нужно ровно 1!

# Слайд 5: Система частиц (Particles) ✨
Частицы — это крошечные спрайты, которые живут доли секунды и создают красоту: искры, дым, огонь, снег.

Сначала описываем **как выглядят** частицы:
```python
# 5. Партиклы на прыжок (искры)
self.jump_emitter = s.ParticleEmitter(s.ParticleConfig(
    amount=8,                         # Сколько частиц за раз
    size_range=(3, 6),                # Размер от 3 до 6 пикселей
    speed_range=(60, 140),            # Скорость разлёта
    angle_range=(160, 200),           # Летят назад-вниз
    lifetime_range=(0.2, 0.5),        # Живут 0.2-0.5 секунды
    fade_speed=400,                   # Скорость затухания
    colors=[(255, 255, 100), (255, 200, 50)],  # Жёлтые искры
))
```

# Слайд 6: Выпуск частиц при прыжке
Всего одна строка! При нажатии прыжка вызываем `emit()`:

```python
if s.input.was_pressed(s.pygame.K_SPACE) or s.input.was_mouse_pressed(1):
    self.player_body.velocity.y = -350
    # Выпускаем искры при прыжке!
    self.jump_emitter.emit(self.player.position)
```

`emit(позиция)` — стреляет частицами из указанной точки. Мы стреляем из позиции птички!

# Слайд 7: Наклон птички 🐦
В настоящей Flappy Bird птичка наклоняет клюв вверх при прыжке и вниз при падении. Это делает анимацию живой!

Добавляем в `update` (после управления, до столкновений):
```python
# --- Наклон птички по скорости ---
self.player.angle = max(-30, min(30, -self.player_body.velocity.y / 12))
```

**Как это работает?**
- Летим вверх → `velocity.y` отрицательная → угол положительный → клюв вверх ↗
- Падаем → `velocity.y` положительная → угол отрицательный → клюв вниз ↘
- `max(-30, min(30, ...))` — ограничиваем наклон, чтобы птичка не крутилась как вертолёт!

# Слайд 8: Тряска камеры при столкновении! 📳
Когда птичка врезается, хочется почувствовать удар! Для этого в SpritePro есть `s.shake_camera`:

```python
def trigger_game_over(self):
    if self.is_game_over:
        return
    self.is_game_over = True
    self.player_body.velocity.y = 0
    
    # Тряска камеры при смерти!
    s.shake_camera(strength=(10, 10), duration=0.3)
    
    # ... текст GAME OVER ...
```

Одна строка — а какой эффект! Профессиональные разработчики называют это **Game Feel** (ощущение от игры).
