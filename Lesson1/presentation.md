# Слайд 2: О библиотеке SpritePro
Чтобы создать хит в рекордные сроки, мы используем **SpritePro** — современный высокоуровневый 2D-движок.
Этот движок написан преподавателем Алгоритмики специально для создания крутых проектов! Он является **мультиплатформенным**, поддерживает **мультиплеер** для сетевых игр и имеет **открытый исходный код** (open source).
Вы можете изучить его код и проект на GitHub: [https://github.com/NeoXider/SpritePro](https://github.com/NeoXider/SpritePro)

Он берёт на себя всю рутину! Нам больше не нужно писать длинные циклы `while`, отслеживать каждый кадр или вручную перерисовывать задний фон (как это было в чистом Pygame).
Движок построен так, чтобы мы могли использовать все свои силы сразу на механику игры — гравитацию, трубы, меню и подсчет очков.

# Слайд 3: Установка и Импорт библиотеки
Перед тем как начать писать код, нам необходимо установить саму библиотеку! Открываем терминал и пишем:
```bash
pip install spritePro
```
Только после успешной установки мы можем подключить библиотеку в нашем главном файле. 
Для этого мы используем конструкцию `import ... as ...` (создание псевдонима):

```python
import spritePro as s
```
Мы пишем `as s`, чтобы каждый раз не печатать длинное слово `spritePro`. Теперь весь мощный функционал доступен нам просто по нажатию одной буквы `s`!

# Слайд 4: Запуск программы (`s.run`)
Чтобы на экране появилось окно игры, в SpritePro требуется вызвать всего одну функцию — `s.run()`. Эта функция запускает всё ядро движка.
У неё есть полезные параметры:
- `size` — размеры открываемого окна (ширина и высота).
- `title` — название игры, которое появится в самом верху окна.

```python
# Так выглядит запуск пустого окна размером 400x600 пикселей:
s.run(size=(400, 600), title="Flappy Bird")
```

# Слайд 5: Создание спрайта (`s.Sprite`)
Всё, что выводится на экран (персонаж, земля, фон, трубы) — это **Спрайт** (`s.Sprite`).
Для того чтобы картинка появилась в игре, мы создаем спрайт, указывая её название и свойства:
- `size=(w, h)` — какую ширину и высоту мы хотим задать картинке.
- `pos=(x, y)` — позиция картинки на экране.

**ОЧЕНЬ ВАЖНО:** По умолчанию в мире SpritePro мы пишем координаты `pos` для **ЦЕНТРА** спрайта! То есть `(200, 300)` поставит именно середину картинки в эту точку, а не её левый верхний угол.

# Слайд 6: Практика: выводим картинки на экран!
Давайте попрактикуемся и выведем фон с птичкой. Вспомним, что размер окна — 400x600 пикселей, а координаты мы задаем для центра. Значит, чтобы фон встал идеально посередине, нам надо указать `pos=(200, 300)`.

```python
import spritePro as s

# Рисуем фон (строго по центру экрана 400х600)
bg = s.Sprite("background.png", pos=(200, 300), size=(400, 600))

# Рисуем птичку (левее по координате X)
player = s.Sprite("bird.png", pos=(100, 300), size=(50, 50))

s.run(size=(400, 600), title="Flappy Bird")
```

# Слайд 7: Порядок отрисовки (Z-index)
А что будет, если в коде сначала создать птичку, а потом фон? Фон просто **перекроет** птичку, и мы её не увидим!
Движок рисует объекты по очереди. У нас есть **два способа** контролировать порядок:
1) **Порядок строк в коде:** Писать вещи, которые дальше (фон), выше по коду, а те, что ближе к экрану (птичка) — ниже (как слои в фотошопе).
```python
bg = s.Sprite("background.png", pos=(200, 300))
player = s.Sprite("bird.png", pos=(100, 300)) # Птичка поверх фона
```

2) **Параметр `sorting_order`:** Можно передать в Спрайт специальный параметр. Чем больше цифра, тем ближе картинка к нам, перекрывая другие:
```python
# Птичка будет поверх фона, даже если написана в коде раньше!
player = s.Sprite("bird.png", pos=(100, 300), sorting_order=10)
bg = s.Sprite("background.png", pos=(200, 300), sorting_order=1)
```

# Слайд 8: Удобные переменные в движке
Чтобы постоянно не высчитывать на калькуляторе математику (например, где находится ровный центр нашего окна `400x600`), разработчики движка добавили удобные переменные-помощники:
- `s.WH` (Width & Height) — содержит размер нашего экрана `[Width, Height]`.
- `s.WH_C` (Width & Height Center) — содержит точные координаты центра экрана `[Width/2, Height/2]`.

# Слайд 9: Применение переменных (Пишем код!)
Теперь мы применим эти переменные на практике! 
Сделаем фон ровно по центру экрана и растянем его на весь размер окна.

```python
import spritePro as s

# Фон ставим точно в центр (s.WH_C) и растягиваем на весь экран (s.WH)
bg = s.Sprite("background.png", pos=s.WH_C, size=s.WH)

# Птичку ставим левее от центра:
player = s.Sprite("bird.png", pos=(100, 300), size=(50, 50))

s.run(size=(400, 600), title="Flappy Bird")
```
*Запустите этот код, и вы увидите окно с фоном и игроком!*

# Слайд 10: Что такое Сцена (Scene)?
Картинки появились, но если мы продолжим писать код плоским текстом, мы не сможем управлять игрой каждый кадр. 
Для того чтобы игра ожила, код всегда структурируют через **Сцены** (`s.Scene`).
Сцена — это площадка (отдельный уровень, главное меню игры, или экран смерти). Именно внутри сцены работает игровой цикл, где каждый объект живет по своим правилам.

# Слайд 11: Переносим код в Конструктор Сцены!
Для перехода на Сцены мы пишем свой класс `GameScene`. Внутри него есть `__init__` (запускается при старте).
**Важно:** теперь всем своим спрайтам нужно передавать параметр `scene=self`, чтобы картинка знала, в какой именно сцене она находится!

```python
class GameScene(s.Scene):
    def __init__(self):
        super().__init__()
        # ВАЖНО: Добавлено scene=self!
        self.bg = s.Sprite("background.png", pos=s.WH_C, size=s.WH, scene=self)
        self.player = s.Sprite("bird.png", pos=(100, 300), size=(50, 50), scene=self)

if __name__ == '__main__':
    # Теперь передаем нашу Сцену в функцию запуска:
    s.run(scene=GameScene, size=(400, 600), title="Flappy Bird: Урок 1")
```

# Слайд 13: Гравитация и Физика (`s.physics`)
В SpritePro есть встроенный мощный физический движок `s.physics`. 
Чтобы птичка начала падать, нужно сделать две вещи:
1. Превратить её в динамическое "тело" через `add_physics`.
2. Включить **гравитацию** — силу, которая тянет всё вниз!

```python
# Пишем это внутри def __init__(self):
self.player_body = s.add_physics(self.player, s.PhysicsConfig(bounce=0.0))

# Устанавливаем гравитацию — без этого птичка не будет падать!
s.physics.set_gravity(980)
```
`bounce=0.0` — чтобы птичка не отскакивала как резиновый мячик. А `980` — это ускорение свободного падения, как в реальности!

# Слайд 14: Физические границы экрана
Если птичка начнет падать, она провалится в бесконечную пустоту под экраном.
Движок позволяет одной строкой создать невидимые стены ровно по границам нашего графического окна!
```python
# Пишем это внутри def __init__(self) сразу под гравитацией:
s.physics.set_bounds(s.pygame.Rect(0, 0, 400, 600))
```
Теперь птичка упадёт и ровно ляжет на пол!

# Слайд 15: Опрос системы ввода (`s.input`)
Чтобы птичка летела, мы должны дать игроку управление. Для этого существует модуль `s.input`, который постоянно следит за вашей клавиатурой и мышкой.
У Сцены есть регулярный метод `update(self, dt)`, который вызывается много раз в секунду. Именно там мы ловим нажатия!

**Что умеет `s.input`?**
- `s.input.was_pressed(s.pygame.K_SPACE)` — фиксация одиночного нажатия на любую клавишу (в данном случае Пробел).
- `s.input.was_mouse_pressed(1)` — фиксация клика мышкой. 
  *Цифра означает кнопку: (1 - левая, 2 - колесико, 3 - правая).*
- `s.input.mouse_pos` — если бы мы делали шутер, здесь всегда лежат точные координаты курсора мыши `[x, y]` в нашем окне!

# Слайд 16: Прыжок птички (Итог Урока)
Дадим игроку выбор! Объединим клавиатуру и мышь с помощью логического оператора `or` (ИЛИ).
Если происходит нажатие, мы задаём физическому телу птички резкую отрицательную скорость по оси Y (летим вверх!).

```python
    def update(self, dt):
        # Если нажат Пробел ИЛИ Левая кнопка мыши
        if s.input.was_pressed(s.pygame.K_SPACE) or s.input.was_mouse_pressed(1):
            
            # Придаем резкую скорость по Y вверх (-400)
            self.player_body.velocity.y = -400
```
Запускайте игру! Птичка падает, и по клику послушно подпрыгивает!
