## ЧАСТЬ 1: Быстрый старт и Спрайты (50 мин)

### 1) Введение и О Библиотеке
Расскажите детям, что мы начинаем создавать настоящий хит — игру Flappy Bird! Подчеркните, как **круто изучать новые библиотеки**: это даёт "сверхспособности", позволяя использовать профессиональные подходы из игровой индустрии всего парой строчек кода.
Для нашего хита мы берём **SpritePro** — современный высокоуровневый 2D-движок.
**Интересный факт для учеников:** этот движок написан преподавателем Алгоритмики специально для создания крутых проектов! Он является **мультиплатформенным**, поддерживает **мультиплеер** для сетевых игр и имеет **открытый исходный код** (open source). Вы можете показать им исходники на GitHub: `https://github.com/NeoXider/SpritePro`
Объясните классный плюс движка: мы избавлены от рутины. Нам больше не нужно писать классический длинный цикл перерисовки кадров `while True` или высчитывать дельту времени. Движок полностью берет закулисную работу на себя!

### 2) Установка и Импорт библиотеки
Напомните, что перед тем как писать код, библиотеку нужно обязательно установить! Попросите детей открыть терминал и прописать команду:
```bash
pip install spritePro
```
Убедившись, что у всех прошла установка, мы подключаем её в самом файле кода. Очень важно объяснить конструкцию `as`:
```python
import spritePro as s
```
**Суть:** `as s` — это создание псевдонима. Разработчикам не хочется постоянно писать длинное слово `spritePro`, поэтому мы "переименовываем" его под короткую букву `s`, через которую будем обращаться к модулям движка.

### 3) Запуск программы (`s.run`)
Весь запуск ядра игры сводится к одной функции:
```python
s.run(size=(400, 600), title="Flappy Bird")
```
- Рассмотрите с детьми параметр `size` (отвечает за ширину и высоту игрового окна в пикселях).
- Рассмотрите `title` — это текст, который игрок будет видеть в шапке окна.

### 4) Что такое Спрайт (`s.Sprite`)
Спрайт — это базовая визуальная сущность в 2D играх. Игрок, враги, даже задний фон — всё это спрайты.
Покажите им параметры:
- `size=(w, h)` — ширина и высота картинки.
- `pos=(x, y)` — позиция на экране.
🔴 **АКЦЕНТИРУЙТЕ ВНИМАНИЕ:** Важнейшее отличие SpritePro состоит в том, что координаты для `pos` указывают на **ЦЕНТР** рисунка (в Pygame они бы указывали на левый верхний угол). Это кардинально меняет логику позиционирования, делая её удобнее и интуитивнее.

### 5) Практика: выводим картинки на экран!
Попросите учеников написать код и вывести фон и птичку вручную.
Так как размеры окна 400x600, центр находится на `(200, 300)`.
```python
import spritePro as s

# Рисуем фон строго по центру 400х600
bg = s.Sprite("background.png", pos=(200, 300), size=(400, 600))
player = s.Sprite("bird.png", pos=(100, 300), size=(50, 50))

s.run(size=(400, 600), title="Flappy Bird")
```

### 6) Порядок отрисовки (Z-index)
Спросите учеников: *"Что будет, если мы сначала создадим птичку, а потом фон?"* 
Объясните, что фон полностью закроет птичку!
Движок отрисовывает элементы слой за слоем. Вы можете контролировать этот порядок двумя способами:
1. **Порядок команд** — кто создается позже в коде (строка ниже), тот рисуется поверх старых изображений:
```python
bg = s.Sprite("background.png", pos=(200, 300))
player = s.Sprite("bird.png", pos=(100, 300)) # Птичка поверх фона
```
2. **Параметр `sorting_order`** — мы можем передать в создание Спрайта `sorting_order=число`. Чем больше число, тем ближе объект к нашему глазу (например, `sorting_order=10` будет поверх `sorting_order=0`):
```python
# Птичка будет сверху, хотя и написана раньше в коде!
player = s.Sprite("bird.png", pos=(100, 300), sorting_order=10)
bg = s.Sprite("background.png", pos=(200, 300), sorting_order=1)
```

### 7) Удобные переменные в движке
Так как математические подсчёты центров экрана часто вызывают трудности (как мы только что считали центр `200, 300`), разработчики SpritePro добавили глобальные константы (переменные для чтения):
- `s.WH` — список, хранящий размеры ширины и высоты (Width/Height) окна (в нашем случае `(400, 600)`).
- `s.WH_C` — список, хранящий точные значения для центра (Center) экрана `(200, 300)`.

### 8) Применение переменных
Давайте напишем наш первый "плоский" (без классов) рабочий код с использованием этих переменных. Бэкграунд будет растянут на `s.WH` и помещен точно в `s.WH_C`.
```python
import spritePro as s

# Загружаем графику с учетом системных переменных движка
bg = s.Sprite("background.png", pos=s.WH_C, size=s.WH)
player = s.Sprite("bird.png", pos=(100, 300), size=(50, 50))

# Запускаем игру
s.run(size=(400, 600), title="Flappy Bird")
```
*(Дети запускают скрипт — на экране появляется картинка!).*

### 9) Сцена (`s.Scene`)
Птичка висит на экране без движения. "Плоский" код хорош для проверки картинок, но в реальных играх нужно обновление каждый кадр. 
Объясните, что вся логика реальной игры должна существовать внутри **Сцены**.
Что такое Сцена? Это как съемочная площадка: Уровень 1, Главное Меню, или Экран настроек — это всё отдельные сцены, на которых разворачиваются свои события. 

### 10) Перетаскиваем код во внутрь Конструктора Сцены (с `scene=self`)
Делаем рефакторинг. Картинки теперь должны создаваться внутри `def __init__(self):` класса Сцены.
Обязательно обратите внимание на новый важный атрибут: `scene=self`. Зачем он? Картинка должна чётко понимать, в какой именно сцене она сейчас отрисовывается.
```python
class GameScene(s.Scene):
    def __init__(self):
        super().__init__()
        # Прикрепляем спрайты к нашей Сцене!
        self.bg = s.Sprite("background.png", pos=s.WH_C, size=s.WH, scene=self)
        self.player = s.Sprite("bird.png", pos=(100, 300), size=(50, 50), scene=self)

if __name__ == '__main__':
    # Теперь мы передаем наш класс в s.run
    s.run(scene=GameScene, size=(400, 600), title="Flappy Bird")
```

## ЧАСТЬ 2: Физика и Ввод данных (30 мин)

### Гравитация и свойства тела
Для того чтобы всё двигалось, нам не нужно писать вручную изменения Y каждую секунду. У нас под рукой `s.physics`.
Мы должны сообщить движку, что птичка теперь — физический объект. Делается это функцией `s.add_physics()`.

Мы также воспользуемся `s.PhysicsConfig`. Попросите детей задуматься: что будет, если птичка упадёт? Она может отпрыгнуть от земли. Нам это не нужно.
```python
        # Отключаем отскок: bounce=0.0
        self.player_body = s.add_physics(self.player, s.PhysicsConfig(bounce=0.0))
```

### Границы Сцены
Если мы запустим игру, птичка упадёт за пределы окна.
Мы можем легко создать невидимые стены, которые оборачивают наш экран, использовав `s.physics.set_bounds`:
```python
        # Устанавливаем прямоугольник границ, который равен нашему экрану
        s.physics.set_bounds(s.pygame.Rect(0, 0, 400, 600))
```

### Система ввода и Прыжок (`s.input`)
В `GameScene` есть метод `update`, который сам вызывается каждый кадр (вспоминаем почему Сцены это удобно!).
Нам нужно поймать одиночное нажатие мыши или клавиатуры! Для этого есть модуль `s.input`.

Расскажите детям, что `s.input` хранит всё состояние клавиатуры и мыши:
- `s.input.mouse_pos` — вернет точные координаты `[x, y]` курсора (полезно для шутеров с прицелом!).
- `s.input.was_mouse_pressed(кнопка)` — вернет `True` в момент клика. Кнопки мыши: **1** — левая, **2** — колесико, **3** — правая.
- `s.input.was_pressed(клавиша)` — вернет `True` в момент нажатия кнопки клавиатуры.

Используем логический оператор `or`, чтобы птичка прыгала и от пробела, и от левого клика мышки:
```python
    def update(self, dt):
        # s.pygame.K_SPACE - пробел, а 1 - это левая кнопка мышки.
        if s.input.was_pressed(s.pygame.K_SPACE) or s.input.was_mouse_pressed(1):
            
            # Делаем резкий импульс в отрицательную сторону (влетаем наверх!)
            self.player_body.velocity.y = -400
```
