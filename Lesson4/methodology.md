## 👨‍🏫 Конспекты для преподавателя

### Часть 1: Сохранение рекорда (save/load)
**Объяснение:** Когда игра закрывается, все переменные пропадают. Чтобы сохранить лучший результат, нужно записать его в файл. SpritePro предоставляет простой API:
- `s.save_load.save({"best": 10}, "best_score.json")` — сохраняет словарь в JSON-файл.
- `s.save_load.load("best_score.json", default_value={"best": 0})` — загружает данные. Если файла нет — возвращает значение по умолчанию.

**Когда сохраняем?** Только при установке нового рекорда — в `trigger_game_over`, если `self.score > self.best_score`.
**Когда загружаем?** В `__init__` — один раз при создании сцены.

### Часть 2: Партиклы на смерть
**Объяснение:** Второй эмиттер с другими настройками. Больше частиц (25), разлетаются на 360°, красные оттенки. Вызывается один раз при столкновении.
Это показывает, что можно иметь **сколько угодно эмиттеров** с разными настройками — каждый для своего эффекта.

### Часть 3: Tween-анимации
**Объяснение:** Tweens (от «in-between» — промежуточный) — это плавные переходы свойств спрайта. Мы используем два:
1. `s.tween_punch_scale(self.score_text, strength=0.3, duration=0.2)` — текст «подпрыгивает» в размере при каждом новом очке. Создаёт ощущение награды.
2. `s.tween_alpha(self.player, to=80, duration=0.5)` — птичка плавно становится полупрозрачной при смерти. Усиливает ощущение проигрыша.

**Game Feel:** Профессиональные разработчики говорят, что 80% «сочности» игры — это обратная связь: тряска, пульсация, вспышки, звуки. Мы уже используем 4 вида обратной связи!

### Часть 4: Звуки и Музыка
**Объяснение:** Игра без звука кажется неестественной. SpritePro содержит встроенный аудиоменеджер.
- `s.audio_manager.play_music("имя.mp3")` — включает фоновую музыку, она зациклится.
- `s.load_sound("имя", "путь.wav")` — загружает звук в кэш.
- `.play()` — воспроизводит короткий эффект (нажатие прыжка, удар).

Звуки можно загрузить один раз при `__init__`, а вызывать уже в `update()` и при смерти птицы.


### Часть 5: Экран нового рекорда
**Объяснение:** При game over проверяем `self.score > self.best_score`. Если да — показываем золотой текст «🏆 НОВЫЙ РЕКОРД!». Это мотивирует игрока попробовать ещё раз!
Цвет `(255, 215, 0)` — это золотой. Используем условные переменные `is_new_record` для чистоты кода.

## Возможные проблемы у учеников
- **Файл best_score.json не создаётся:** save_load создаёт файл рядом со скриптом. Убедитесь, что запускаете из папки Lesson4.
- **default_value не работает:** Проверьте, что передаётся именно словарь `{"best": 0}`, а не число.
- **Tween не видно:** `tween_punch_scale` срабатывает быстро (0.2 сек). Увеличьте strength для демонстрации.
- **Эмоджи не отображаются:** Зависит от шрифта. Можно заменить на текст «НОВЫЙ РЕКОРД!».
